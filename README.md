# AI Hiring Bias Detection System

> Detecting, measuring, and mitigating bias in AI-driven recruitment — promoting fair, transparent, and responsible hiring.

---

## 📌 Overview

This project builds an AI-powered resume screening system and rigorously analyses bias in automated hiring decisions. Using a synthetic candidate dataset of 100 profiles, it walks through the full pipeline — from raw data creation to fairness-constrained shortlisting — making it ideal for understanding how unchecked algorithms can systematically disadvantage candidates based on gender, institutional background, or experience.

The project is structured as a two-part lab:

- **Part 1** — Detect bias manually using selection rates and disparate impact metrics
- **Part 2** — Apply fairness constraints to the scoring model and compare outcomes

---

## Objectives

- Simulate a realistic AI hiring pipeline with intentional bias embedded in the scoring logic
- Detect gender bias, institutional bias (college tier), and experience bias
- Compute fairness metrics including selection rates and disparate impact ratios
- Apply and evaluate fairness constraints to produce more equitable shortlists
- Visualize and interpret the impact of bias mitigation

## 📊 Dataset

The synthetic dataset contains **100 candidate profiles** with the following attributes:

| Column | Description |
|---|---|
| `Candidate_ID` | Unique identifier (C1–C100) |
| `Gender` | Male / Female |
| `College` | Tier1 / Tier2 / Tier3 |
| `Experience` | Years of experience (1–6) |
| `Skills_Score` | Score between 55–95 |
| `Selected` | Baseline hiring decision (Yes/No) |
| `Experience_Group` | Bucketed: Low / Medium / High |
| `Baseline_Score` | Biased composite score |
| `Baseline_Rank` | Rank based on biased score |
| `Constraint1/2/3_Score` | Scores after applying fairness constraints |
| `Fair_Rank` | Final rank after fairness correction |
| `Fair_Shortlisted` | Shortlist decision after bias mitigation |
| `Baseline_Shortlisted` | Shortlist decision before bias mitigation |

> **Note:** The baseline scoring intentionally embeds bias — male candidates and Tier 1 college graduates receive hidden score boosts — to simulate real-world algorithmic discrimination.

## Bias Detection Approach

### Metrics Used

**Selection Rate**
```
Selection Rate = Selected Candidates in Group / Total Candidates in Group
```

**Disparate Impact**
```
Disparate Impact = Selection Rate (Disadvantaged Group) / Selection Rate (Reference Group)
```
A disparate impact score below **0.80** signals a potential fairness concern (the "80% rule").

### Groups Analysed

| Dimension | Reference Group | Compared Against |
|---|---|---|
| Gender | Male | Female |
| College | Tier 1 | Tier 2, Tier 3 |
| Experience | High Experience | Low, Medium |

## Key Findings (Sample)

- Male candidates received a **+5 point hidden boost** in the baseline scoring model
- Tier 1 college graduates received a **+6 point advantage** over Tier 3 candidates
- Disparate impact analysis revealed selection rate gaps that breach the 80% threshold
- After applying fairness constraints, shortlist composition shifted measurably toward underrepresented groups
